- [[추상메서드]]를 가지고 있으면 반드시 추상 클래스로 선언해야한다.
- 추상클래스 선언은 abstract키워드로 한다.

>[!warning]
>추상 클래스는 객체를 생성할 수 없다.

- 추상메서드를 가진 추상클래스는 동작이 정의되어 있지 않으므로 인스턴스를 생성할 수 없다.
- 추상클래스는 먼저 상속을 통해 자식클래스를 만들고 ==자식클래스에서 추상메서드를 [[Overriding]]해야 인스턴스를 생성==할 수 있다.
- 추상클래스의 객체를 생성하는 코드는 컴파일 오류가 발생한다.


추상클래스 상속
=
- 추상클래스를 상속받는 자식클래스는 추상클래스가 된다.
- 추상클래스의 추상메서드를 그대로 상속받기 때문에==오버라이딩==을 하지 않으면 자식클래스에 abstract 를 붙어 추상클래스임을 명시해야 컴파일 오류가 발생하지 않는다.

추상클래스 구현목적
= 
- 추상메서드가 포함된 클래스를 상속받는 자식클래스가 반드시 ==오버라이딩==하여 구현하기 위함이다.
- 추상클래스는 추상메서드를 통해 자식클래스가 구현할 메서드를 명료하게 알려주는 인터페이스 역할을 한다.
- 자식클래스는 추상메서드를 목정에 맞게 구현하는 [[객체지향의 4대 특성|다형성]]을 실현한다.
