상속의 개념
- 부모클래스의 멤버(필드, 메서드, 이너클래스)를 자식클래스가 내려받아(상속) 클래스 내부에 포함

상속의 장점
- 코드의 중복성 제거
- ==다형적 표현 가능(다형성)==

상속 문법

1. extends 키워드 사용

		class 자식클래스 extends 부모클래스 {}
2. 다중 상속 불가

		class 자식클래스 extends 부모클래스,부모클래스 {} 
3. 부모클래스가 둘이 될 수는 없지만 자식클래스는 여러 개 가능


>[!important]
>⭐자식은 반드시 부모생성자를 책임져야한다.
>⭐상속이 진행이 되면 부모의 모든 것은 자식것이다. (생성자, private 제외)
>⭐부모객체참조변수는 모든 자식 객체를 저장할 수 있다.([[객체의 타입변환|업캐스팅]] : 묵시적변환)
>⭐자식객체참조변수는 부모객체를 저장하는 경우에는 두가지 경우가있다.
>	참조) [[객체의 타입변환|다운캐스팅]]
>⭐자바는 다중상속이 되지 않는다. (멤버변수나 멤버함수가 똑같으면 누구것인지 ==모호성==을 발생시기기때문)
>⭐상속에서 발생되는 오버라이딩은 인스턴스멤버 함수에만 적용이된다.
>	(인스턴스필드,정적멤버필드,정적멤버함수는 메모리에 각각 다른위치에 존재하기때문에 적용이 되지 않는다.)



Super 키워드
=
- 필드명 중목 또는 메서드 오버라이딩으로 가려진 부모의 필드/메서드를 호출하기 위해 주로 사용

Super() 메서드
=
- 부모 클래스의 생성자를 호출
- super() 메서드는 ==생성자 내부에서만 사용==가능
- 반드시 중괄호 이후 ==첫 줄에 위치==하여야 함
- 자식클래스 생성자의 ==첫 줄에는 반드시 this() 또는 super()가 포함==되어야 함
	(생략시 컴파일러가 자동으로 super()추가)
